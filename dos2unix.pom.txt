NOTE: this function is preprocessed out in the build in the Redemption container, so it need not be included here.

While dirname() and basename() are defined in common.c, they are ifdef'd out by the Redemption container.

POSIX says: The dirname() function may modify the string pointed to by path, and may return a pointer to static storage that may then be overwritten by a subsequent call to dirname().
and says the same wrt basename().  So assume dirname() and basename() take & return irresponsible pointers, just like their sample implementations in common.c

Note that strdup() takes a valid irresponsible pointer, and returns a responsible pointer that is GOOD or NULL.

  int ResolveSymbolicLink(char *lFN, char **rFN, CFlag *ipFlag, const char *progname)

      parameters:
        lFN                irresponsible, VALID
        *rFN               producer
        *rFN               responsible    ZOMBIE|NULL->GOOD|unchanged&error (return -1)
        ipFlag             irresponsible, VALID
        progname           irresponsible, VALID
      locals:
        errstr             irresponsible
        targetFN           responsible
      return:
        n/a

If lFN is a link, and realpath() or canonicalize_file_name() fail, they both return NULL, but the null check does not overrite *rFN. So if this function returns -1 as error, it leaves *rFN unchanged.

If lFN is a link, then canonicalize_file_name() returns a responsible pointer. (So does realpath().) If GOOD, it gets assigned to *rFN and then goes out of scope. Which means that *rFN must be responsible.
Otherwise lFN is not a link, it gets directly assigned to *rFN.  This would mean either that *rFN is irresponsible (which contradicts lFN being a link) or that *rFN is responsible and lFN becomes a ZOMBIE.
That is, lFN becomes a ZOMBIE only if it is not a symbolic link.
So this function violates POM!

A simple fix would be for *rFN to be responsible (ZOMBIE|NULL->GOOD) and receive a clone of lFN, perhaps by using strdup(lFN) on line 1100 and 1102. (This would make lFN irresponsible).  This function is invoked only by ConvertNewFile(), which treats lFN as irresponsible and *rFN as responsible.

TargetFN should be responsible, But line 1513 violates TargetFN being responsible, by giving it ipOutFN (which is irresponsible).
This could be fixed by:
comment out line 1513 (TargetFN must remain a ZOMBIE),
add:
    else {
      TargetFN = strdup(ipOutFN);
    }
to line 1526
and make the freeing of TargetFN unconditional (remove the condition)

Line 1515 says:
    ResolveSymlinkResult = 0; /* indicates that TargetFN need not be freed */
and further analysis shows that the comment is correct: the ResolveSymlinkResult flag does correctly indicate whether TargetFN should be freed.
So this code is currently memory-safe, but using a variable to keep track of whether TargetFN should be freed violates POM.
